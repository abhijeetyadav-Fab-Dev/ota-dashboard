# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common Development Commands

- **Start development server**: `npm run dev` (or `yarn dev`). Runs Next.js in dev mode with hot‑reloading.
- **Build for production**: `npm run build`. Compiles the app and generates the `.next` output.
- **Run production server**: `npm start`. Starts the built app (`next start`).
- **Lint code**: `npm run lint`. Uses ESLint with the project's configuration.
- **Type‑check**: `npx tsc --noEmit`. Verifies TypeScript types without emitting files.
- **Prisma generate**: `npx prisma generate`. Regenerates the Prisma client after schema changes.
- **Prisma migrate**: `npx prisma migrate dev` (or `npx prisma migrate deploy` in production). Runs migrations and updates the SQLite DB.
- **Run tests**: No test script is defined. When a test runner is added, use `npm test` and run a single test file with `npm test -- <path/to/test>`.

## High‑Level Architecture

- **Framework**: Next.js (App Router) with TypeScript. UI and server‑side code live under `app/`.
- **API Layer**: REST‑style API routes in `app/api/`. Each route is a `route.ts` exporting a handler.
- **Authentication**: JWT stored in the `ota_session` cookie; validated by `middleware.ts` except for public paths (`/login`, `/api/auth/login`).
- **Database**: Prisma ORM with a **SQLite** datasource (`prisma/schema.prisma`). The generated client lives in `lib/generated/prisma`.
- **Styling**: Tailwind CSS (`tailwind.config.ts`) and global styles in `app/globals.css`.
- **Utilities**: `bcryptjs` (password hashing), `better-sqlite3` (SQLite driver), `jose` (JWT handling).
- **Runtime**: `next dev` for development, `next start` for production.
- **Deployment**: `ecosystem.config.js` for PM2 and Windows batch/PowerShell scripts (`run-dashboard-service.ps1`, `start-dashboard‑*.bat`).

## Important Files & Directories

- `package.json` – scripts, dependencies, dev dependencies.
- `next.config.ts` – Next.js configuration.
- `middleware.ts` – request‑level authentication guard.
- `prisma/schema.prisma` – database schema and client generation.
- `app/` – UI pages and server components.
- `app/api/` – backend endpoints.
- `lib/generated/prisma/` – autogenerated Prisma client.
- `ecosystem.config.js` – PM2 process manager configuration.

## Notes for Claude Code

- **File placement**: Add UI pages under `app/<section>/page.tsx` and API routes under `app/api/<resource>/route.ts`.
- **TypeScript**: Follow the existing strict TypeScript conventions; run `npx tsc --noEmit` after changes.
- **Prisma workflow**: After any schema change, run `npx prisma generate` (and `npx prisma migrate dev` locally).
- **Environment**: The JWT secret is read from `SESSION_SECRET`. The default placeholder is **not** for production.
- **Testing**: No test framework is currently set up. When adding one, add a `test` script to `package.json` and follow the standard `npm test` pattern.
